Платформа подписок и рассылок

Мультиязычный backend-сервис для настроек подписчика, кодов подтверждения, отписки, массовых рассылок и совместимости с legacy-системой подписок Bravo.

Реализовано

  • Публичные и внутренние Flask routes для настроек подписчика, подтверждения, отписки, миграции и sync-флоу.
  • Одноразовые коды подтверждения на Redis с защитой от повторного использования.
  • Раздельные статусы подписок для русского и иврита с логикой seller mirrors.
  • Единое статусное письмо для включения, отключения и смешанных изменений подписок.
  • Фоновая рассылка super_price с батчингом и логированием отправок.
  • Фильтрация контента по языку, городам, секциям и истории заказов клиента.
  • HTML email templates и выбор домена для русских и ивритоязычных seller-страниц.
  • Legacy-синхронизация и импорт подписчиков, фильтров и истории из старых tickets-таблиц.

Технологии: Python, Flask, MySQL, Redis, REST API, Jinja/HTML Email, threading

Mail Analyzer / Обработка bounce-писем

Внутренний сервис анализа email-ошибок, который обрабатывает Gmail bounce-трафик и формирует кандидатов на отписку.

Реализовано

  • Интеграция Gmail API через OAuth.
  • Асинхронная обработка через отдельный analyze endpoint с блокировкой параллельного запуска.
  • Строгий парсинг MIME/DSN-структур и заголовков X-Failed-Recipients.
  • Извлечение проблемных адресов из заголовков, DSN-частей и тела письма.
  • Классификация кейсов: user unknown, mailbox unavailable, quarantine/spam, inactive email и changed email address.
  • Пошаговый regex-анализ перед fallback на LLM.
  • Формирование JSON-отчетов со статистикой и очисткой данных. OpenAI API использовался как fallback для сложных писем.

Технологии: Python, Gmail API, OAuth, MIME/DSN, Regex, JSON, OpenAI API

Сервис интеграции с Bravo DB

Внутреннее Flask API поверх Bravo MySQL-данных, используемое в сценариях подписок и рассылок.

Реализовано

  • REST endpoints для super price show, seller domains, active sellers, order history и всех email, связанных с документом клиента.
  • Сложные SQL-join'ы по orders, shows, seances, halls, cities, sellers, partners и discount-таблицам.
  • Агрегация метаданных шоу, диапазонов дат, списков городов и discount-флагов для рассылочных кампаний.
  • Получение истории заказов клиента для персонализации по региону и секции.
  • Маппинг продавцов и доменов с mirror ID и статусом активности партнера.
  • Переиспользуемый MySQL gateway с connection pooling, retry-логикой и явным управлением транзакциями.

Технологии: Python, Flask, MySQL, SQL, REST API, connection pooling
