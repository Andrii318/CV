Platforma subskrypcji i wysylek

Wielojezyczny backendowy serwis do ustawien subskrybenta, kodow potwierdzenia, wypisania, masowych wysylek oraz zgodnosci z legacy-systemem subskrypcji Bravo.

Zrealizowano

  • Publiczne i wewnetrzne Flask routes dla ustawien subskrybenta, potwierdzenia, wypisania, migracji oraz sync-flow.
  • Jednorazowe kody potwierdzenia w Redis z ochrona przed ponownym uzyciem.
  • Oddzielne statusy subskrypcji dla rosyjskiego i hebrajskiego z logika seller mirrors.
  • Jeden email statusowy dla wlaczenia, wylaczenia oraz mieszanych zmian subskrypcji.
  • Wysylka super_price w tle z batchingiem i logami wysylek.
  • Filtrowanie tresci wedlug jezyka, miast, sekcji oraz historii zamowien klienta.
  • HTML email templates oraz wybor domeny dla rosyjskich i hebrajskich seller pages.
  • Legacy-synchronizacja oraz import subskrybentow, filtrow i historii ze starych tickets tables.

Technologie: Python, Flask, MySQL, Redis, REST API, Jinja/HTML Email, threading

Mail Analyzer / Bounce Processing

Wewnetrzny serwis analizy bledow email, ktory przetwarza Gmail bounce traffic i tworzy kandydatow do wypisania.

Zrealizowano

  • Integracja Gmail API przez OAuth.
  • Asynchroniczne przetwarzanie przez osobny analyze endpoint z blokada rownoleglego uruchomienia.
  • Scisle parsowanie struktur MIME/DSN oraz naglowkow X-Failed-Recipients.
  • Ekstrakcja problematycznych adresow z naglowkow, czesci DSN oraz tresci wiadomosci.
  • Klasyfikacja przypadkow: user unknown, mailbox unavailable, quarantine/spam, inactive email oraz changed email address.
  • Etapowa analiza regex przed fallbackiem do LLM.
  • Generowanie raportow JSON ze statystykami i cleanupem danych. OpenAI API bylo fallbackiem dla trudnych emaili.

Technologie: Python, Gmail API, OAuth, MIME/DSN, Regex, JSON, OpenAI API
