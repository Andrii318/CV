Проєкти

Платформа підписок і розсилок

Мультимовний backend-сервіс для налаштувань підписника, кодів підтвердження, відписки, масових розсилок і сумісності з legacy-системою підписок Bravo.

Реалізовано

  • Публічні та внутрішні Flask routes для налаштувань підписника, підтвердження, відписки, міграції та sync-флоу.
  • Одноразові коди підтвердження на Redis із захистом від повторного використання.
  • Окремі стани підписок для російської та івриту з логікою seller mirrors.
  • Єдиний статусний лист для увімкнення, вимкнення та змішаних змін підписок.
  • Фонова розсилка super_price з батчингом і логуванням відправок.
  • Фільтрація контенту за мовою, містами, секціями та історією замовлень клієнта.
  • HTML email templates і вибір домену для російськомовних та івритомовних seller-сторінок.
  • Legacy-синхронізація та імпорт підписників, фільтрів і історії зі старих tickets-таблиць.

Технології: Python, Flask, MySQL, Redis, REST API, Jinja/HTML Email, threading

Mail Analyzer / Обробка bounce-листів

Внутрішній сервіс аналізу email-помилок, який обробляє Gmail bounce-трафік і формує кандидатів на відписку.

Реалізовано

  • Інтеграція Gmail API через OAuth.
  • Асинхронна обробка через окремий analyze endpoint із блокуванням паралельного запуску.
  • Строгий парсинг MIME/DSN-структур і заголовків X-Failed-Recipients.
  • Витягування проблемних адрес із заголовків, DSN-частин і тіла листа.
  • Класифікація кейсів: user unknown, mailbox unavailable, quarantine/spam, inactive email і changed email address.
  • Покроковий regex-аналіз перед fallback на LLM.
  • Формування JSON-звітів зі статистикою та очищенням даних. OpenAI API використовувався як fallback для складних листів.

Технології: Python, Gmail API, OAuth, MIME/DSN, Regex, JSON, OpenAI API

Сервіс інтеграції з Bravo DB

Внутрішнє Flask API поверх Bravo MySQL-даних, яке використовувалося у сценаріях підписок і розсилок.

Реалізовано

  • REST endpoints для super price show, seller domains, active sellers, order history і всіх email, пов'язаних із документом клієнта.
  • Складні SQL-join'и по orders, shows, seances, halls, cities, sellers, partners і discount-таблицях.
  • Агрегація метаданих шоу, діапазонів дат, списків міст і discount-прапорців для розсилкових кампаній.
  • Отримання історії замовлень клієнта для персоналізації за регіоном і секцією.
  • Мапінг продавців і доменів із mirror ID та статусом активності партнера.
  • Перевикористовуваний MySQL gateway із connection pooling, retry-логікою та явним керуванням транзакціями.

Технології: Python, Flask, MySQL, SQL, REST API, connection pooling

Сумісність і міграція legacy Bravo

Шар сумісності та міграційні процеси між новим dispatch-сервісом і legacy-екосистемою підписок Bravo/global_bravo.

Реалізовано

  • Best-effort синхронізація назад у legacy-таблиці Subscribe та SubscribeHistory.
  • Збереження та генерація legacy SID-ідентифікаторів для сумісності підписників.
  • Синхронізація міст і секцій між новою та старою схемами.
  • Фоновий імпорт підписників, фільтрів, історії, міст і секцій із legacy tickets-таблиць.
  • Підтримка legacy CRM-флоу та мультимовних subscribe/unsubscribe templates в екосистемі Bravo.
  • Безпечна обробка null-дат і відмінностей між старою та новою схемами БД.

Технології: Python, Flask, MySQL, SQL, data migration, legacy systems
